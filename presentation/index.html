Godot 4.6 / C# / 2D platformer

Save The Princess

Krátká technická ukázka side-scrolling akčního platformeru: jak je postavená hlavní scéna, jak funguje soubojový systém a jak se svět mění z normálního lesa do podzimu a zimy.

Doporučená délka: přibližně 5 minut, 8 slidů.

Na začátku řekni, že jde o prototyp hry vytvořený v Godotu a C#. Prezentace se zaměřuje na strukturu a implementaci, nejen na vizuál.

1. Nápad hry

Hráč ovládá prince, který prochází horizontálním fantasy světem. Bojuje se slizy a strážemi, léčí se pomocí jablek, dojde do zimního tábora, promluví s princeznou, zachrání ji a spustí závěrečnou scénu.

Žánr

2D side-scrolling akční platformer.

Herní smyčka

Pohyb, souboj, léčení a postup do dalšího biomu.

Cíl

Dostat se k princezně a dokončit záchrannou scénu.

Tuhle část nech krátkou. Nejdřív vysvětli hráčský zážitek, až potom kód.

2. Technologie

Herní engine

Godot 4.6, 2D scene tree a zem postavená přes TileMap.

Jazyk

C# skripty řeší pohyb, souboje, zdraví, dialogy a zóny.

Assety

GandalfHardcore sprity pro hráče, nepřátele, biomy, dekorace a HUD.

Hlavní scéna: main_node.tscn. Scéna hráče: character.tscn. Většina herní logiky je ve složce scripts/.

Zmiň, že Godot scény drží nody a exportované hodnoty, zatímco C# skripty definují chování.

3. Architektura scén

  • MainNode obsahuje celý level.
  • Tři CanvasLayer pozadí se plynule prolínají podle zóny.
  • TileMap vytváří zem a kolize pro pohyb.
  • Nepřátelé, jablka, stráže, stany a princezna jsou instance scén.
  • Hráč obsahuje kameru, animátor, hitbox a HUD.

Zjednodušený strom scény

MainNode
├── NormalBG / AutumnBG / WinterBG
├── SnowOverlay
├── TileMap
├── Decor
├── Player
├── SlimeEnemy variants
├── GuardEnemy / SecondGuardEnemy
├── Princess
└── ZoneManager
Vysvětli hlavní myšlenku: kompozice. Hra není jeden obří skript, ale mnoho menších scén a skriptů.

4. Postup biomy

Svět je jeden dlouhý horizontální level rozdělený podle X souřadnice.

Normální les

x = 0-1728
Začátek hry a první nepřátelé.

Podzim

x = 1728-3456
Více dekorací, jablek a nepřátel.

Zima

x = 3456+
Sněžení, stany, stráže a finále s princeznou.

Tady zdůrazni, že level není rozdělený na více scén. Pozadí a zem se mění podle toho, jak hráč postupuje doprava.

5. Kód pro prolínání zón

ZoneManager.cs každý frame čte X pozici hráče. Z ní spočítá dvě přechodové hodnoty a použije je jako alpha hodnoty pro podzimní a zimní vrstvy pozadí.

Díky tomu vznikne plynulý přechod biomů bez načítání nové scény.

scripts/ZoneManager.cs

private const float Zone1End = 1728f;
private const float Zone2End = 3456f;
private const float FadeSize = 256f;

float px = player.GlobalPosition.X;

float t12 = Mathf.Clamp(
  (px - Zone1End + FadeSize * 0.5f) / FadeSize,
  0f,
  1f
);

float t23 = Mathf.Clamp(
  (px - Zone2End + FadeSize * 0.5f) / FadeSize,
  0f,
  1f
);

SetAlpha(_autumnSprites, t12 * (1f - t23));
SetAlpha(_winterSprites, t23);
SetAlpha(_snowSprites, t23);
Nevysvětluj každý řádek. Stačí říct, že t12 je přechod normální les až podzim a t23 je podzim až zima.

6. Ovládání hráče

  • Pohyb funguje přes šipky i WASD.
  • Shift používá sprint energii zobrazenou v HUDu.
  • Útok používá dočasný hitbox před hráčem.
  • Smrt zobrazí restart overlay: Press B to start again.

scripts/PlayerController.cs

private enum State {
  Idle, Move, Run, Jump, Fall,
  Attack, Hurt, Dead
}

if (Input.IsActionJustPressed("attack")) {
  SetState(State.Attack);
}

if (_state == State.Attack &&
    !_hitboxShape.Disabled &&
    !_hasDealtDamageThisAttack) {
  foreach (var body in _attackHitbox.GetOverlappingBodies()) {
    if (body is IDamageable damageable) {
      damageable.TakeDamage(AttackDamage);
      _hasDealtDamageThisAttack = true;
      break;
    }
  }
}
State machine vysvětli jednoduše: hráč může být vždy jen v jednom hlavním stavu. Attack hitbox je aktivní jen v části animace.

7. Nepřátelé, zdraví a dialogy

Kontrakt pro poškození

IDamageable umožňuje hráči, slizům a strážím sdílet stejné API pro poškození.

AI nepřátel

Slizové patrolují, pronásledují hráče, útočí na dosah a umírají při 0 HP.

Interakce

Jablka léčí, stráže mluví nebo bojují, princeznu lze rozvázat a spustit konec hry.

scripts/IDamageable.cs

public interface IDamageable
{
  void TakeDamage(int damage);
}
Tenhle slide je hlavně o znovupoužití. Jednoduchý interface dovolí, aby různé entity reagovaly na útoky stejně.
